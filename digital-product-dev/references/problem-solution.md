# Phase 1: Problem / Solution Fit

## เป้าหมาย
ค้นหาว่า "ปัญหาจริงๆ คืออะไร" และ "solution ของเราแก้ปัญหานั้นได้จริงไหม" ก่อนลงทุนเวลาสร้าง

---

## ขั้นตอนการทำงาน

### Step 1: Define the Problem

ถามผู้ใช้ว่า:
- "คุณเห็นปัญหาอะไรในชีวิตหรือการทำงานของคนกลุ่มหนึ่ง?"
- "ใครกำลังทุกข์ทรมานกับปัญหานี้อยู่?"
- "ตอนนี้คนแก้ปัญหานี้ยังไง? ทำไมถึงยังไม่พอ?"

**Problem Statement Template:**
```
ลูกค้า: [Customer Segment — เจาะจง ไม่ใช่ "ทุกคน"]
ปัญหา: [ปัญหาที่พบ — ใช้ภาษาของลูกค้า ไม่ใช่ภาษาเรา]
สถานการณ์: [เมื่อไหร่/ที่ไหนที่ปัญหานี้เกิด]
ผลกระทบ: [เกิดอะไรขึ้นถ้าไม่มีการแก้ไข — ความเจ็บปวด/ค่าใช้จ่าย/เสียเวลา]
ความถี่: [เกิดบ่อยแค่ไหน — ทุกวัน/ทุกสัปดาห์/เป็นครั้งคราว]
```

**ตัวอย่างที่ดี:**
```
ลูกค้า: Freelance Designer ที่ทำงาน 1-3 คน
ปัญหา: ใช้เวลานานในการสร้าง proposal และ invoice ให้ลูกค้าแต่ละราย
สถานการณ์: หลังได้ project ใหม่ หรือเมื่อถึงเวลาออก invoice
ผลกระทบ: เสียเวลา 2-3 ชั่วโมงต่อครั้ง / ผิดพลาดบ่อย / ดูไม่มืออาชีพ
ความถี่: 2-4 ครั้งต่อเดือน
```

---

### Step 2: Identify Customer Segment

เจาะจง customer segment ให้ชัด — segment ที่ดีต้อง:
- **มีขนาดพอ** — มีพอที่จะสร้างธุรกิจได้
- **เข้าถึงได้** — รู้ว่าจะหาพวกเขาได้ที่ไหน
- **มีปัญหาเร่งด่วน** — พร้อมจ่ายเพื่อแก้ปัญหา

**Customer Persona Template:**
```
ชื่อ Persona: [ชื่อสมมติ]
อาชีพ/สถานการณ์: [บริบทชีวิต]
เป้าหมายหลัก: [อยากได้อะไร]
ความกลัว/ปัญหา: [กลัวอะไร/ทุกข์กับอะไร]
พฤติกรรมปัจจุบัน: [ทำยังไงอยู่ตอนนี้]
ช่องทางที่ใช้: [ใช้ app/tool อะไร ไปที่ไหน]
```

### Step 3: Analyze Current Alternatives

ลูกค้าแก้ปัญหานี้ยังไงอยู่ตอนนี้?

| วิธีแก้ปัจจุบัน | ข้อดี | ข้อเสีย |
|-----------------|-------|---------|
| [วิธี 1] | | |
| [วิธี 2] | | |
| [วิธี 3] | | |

คำถามที่ต้องตอบได้: "ทำไมวิธีเหล่านี้ถึงยังไม่เพียงพอ?"

---

### Step 4: Solution Hypothesis

**Solution Hypothesis Template:**
```
เราเสนอ: [solution ของเรา]
สำหรับ: [customer segment]
เพื่อช่วยให้: [ผลลัพธ์ที่ต้องการ]
ต่างจากคู่แข่งตรงที่: [unique differentiator]
เราเชื่อว่าลูกค้าจะ: [พฤติกรรมที่คาดหวัง]
```

---

### Step 5: Value Proposition

**Value Proposition Canvas (แบบย่อ):**

```
ลูกค้าได้: [Gain — ประโยชน์ที่ได้รับ]
ลดความเจ็บปวด: [Pain — ปัญหาที่แก้ได้]
ด้วย: [Product/Service ของเรา]
```

**Jobs-to-be-Done Statement:**
```
เมื่อฉัน [สถานการณ์]
ฉันต้องการ [งานที่ต้องทำ]
เพื่อ [ผลลัพธ์ที่ต้องการ]
```

## Checklist Phase 1 ✓

- [ ] Problem Statement ชัดเจน ไม่กำกวม
- [ ] Customer Segment เจาะจง (ไม่ใช่ "ทุกคน")
- [ ] รู้ว่าคนแก้ปัญหานี้ยังไงอยู่ (current alternatives)
- [ ] Solution Hypothesis เป็น hypothesis จริงๆ (ยังไม่ใช่ความจริง)
- [ ] Value Proposition ชัดเจนใน 1-2 ประโยค
