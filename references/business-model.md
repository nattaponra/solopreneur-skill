## Lean Canvas (9 Blocks)

ใช้ Lean Canvas แทน Business Model Canvas เพราะเหมาะกับ startup/solo entrepreneur มากกว่า

```
┌─────────────────┬────────────────┬─────────────────┬────────────────┬─────────────────┐
│  PROBLEM        │  SOLUTION      │  UNIQUE VALUE   │  UNFAIR        │  CUSTOMER       │
│                 │                │  PROPOSITION    │  ADVANTAGE     │  SEGMENTS       │
│ ปัญหาหลัก 3 อย่าง│ วิธีแก้หลัก   │                 │                │                 │
│                 │                │ ประโยคเดียวที่  │ สิ่งที่คัดลอก  │ กลุ่มลูกค้า     │
│ Existing        │                │ บอกว่าเราคือใคร │ ไม่ได้ง่ายๆ    │ หลัก            │
│ Alternatives    │                │ และต่างจากคนอื่น│                │                 │
├─────────────────┴────────────────┼─────────────────┼────────────────┴─────────────────┤
│  KEY METRICS                     │  CHANNELS       │                                   │
│                                  │                 │                                   │
│ ตัวชี้วัดที่บอกว่าธุรกิจ         │ ช่องทางเข้าถึง  │                                   │
│ กำลังไปได้ดี                     │ ลูกค้า          │                                   │
├──────────────────────────────────┴─────────────────┴───────────────────────────────────┤
│  COST STRUCTURE                                    │  REVENUE STREAMS                  │
│                                                    │                                   │
│ ค่าใช้จ่ายหลักในการ run ธุรกิจ                    │ แหล่งรายได้และโมเดลการเก็บเงิน    │
└────────────────────────────────────────────────────┴───────────────────────────────────┘
```

### วิธีกรอก Lean Canvas แต่ละ Block

**1. PROBLEM**
- ระบุปัญหาหลัก 1-3 อย่างที่ลูกค้ามี (จาก Phase 1)
- ระบุ "Existing Alternatives" — ตอนนี้ลูกค้าใช้อะไรแก้

**2. CUSTOMER SEGMENTS**
- กลุ่มลูกค้าหลักที่จะ target ก่อน
- ระบุ "Early Adopters" — ใครที่จะซื้อแรกสุด

**3. UNIQUE VALUE PROPOSITION**
- 1 ประโยคที่บอกว่าเราคืออะไรและดีกว่ายังไง
- Format: "เราช่วย [X] ให้ [ทำ Y] โดย [Z]"
- ต้องเป็น message ที่ลูกค้าเข้าใจได้ใน 5 วินาที

**4. SOLUTION**
- 3 features หลักที่แก้ปัญหาจาก block 1
- ยังไม่ต้อง detail — แค่ concept level

**5. CHANNELS**
- ช่องทาง acquire ลูกค้า: Social, SEO, Direct Sales, Partnership
- ช่องทาง deliver product: App, Web, Email, etc.

**6. REVENUE STREAMS**
- โมเดลการหารายได้ (ดูรายละเอียดด้านล่าง)
- ราคาที่คาดว่าจะเก็บได้

**7. COST STRUCTURE**
- ค่าใช้จ่ายหลัก: Dev, Infrastructure, Marketing, Operations
- Fixed vs Variable costs

**8. KEY METRICS**
- 1-3 ตัวเลขที่วัดว่าธุรกิจสุขภาพดีหรือไม่
- เช่น: MRR, Active Users, Churn Rate, NPS

**9. UNFAIR ADVANTAGE**
- สิ่งที่ทำให้เราได้เปรียบที่คัดลอกได้ยาก
- เช่น: Network effect, Brand, Data, Expertise, Community

## Revenue Model Options

เลือก model ที่เหมาะกับ product และลูกค้า:

| Model | เหมาะกับ | ตัวอย่าง |
|-------|----------|---------|
| **Subscription (SaaS)** | B2B tools, recurring value | Notion, Figma |
| **One-time Purchase** | Tools, templates | Envato items |
| **Freemium** | Consumer apps ที่ต้องการ scale | Spotify, Dropbox |
| **Marketplace** | Connecting buyers & sellers | Airbnb, Fiverr |
| **Usage-based** | Variable consumption | AWS, Twilio |
| **Professional Services** | Custom work + software | Agencies |

### การตั้งราคา
- **Value-based**: ตั้งราคาตาม value ที่ลูกค้าได้รับ (แนะนำ)
- **Cost-plus**: ต้นทุน + กำไร
- **Competitive**: ตามคู่แข่ง

## Unit Economics (สำคัญมาก)

### ตัวเลขที่ต้องรู้ก่อน launch

```
CAC (Customer Acquisition Cost) = งบ Marketing / จำนวนลูกค้าใหม่
LTV (Lifetime Value) = ARPU × Average Lifetime
LTV:CAC Ratio = ต้องการ > 3:1 สำหรับธุรกิจที่ healthy
Payback Period = CAC / Monthly Gross Profit per Customer
```

**เป้าหมาย:**
- LTV:CAC > 3:1
- Payback Period < 12 เดือน
- Churn Rate < 5% ต่อเดือน (สำหรับ SaaS)
